 Backend

 routes/: API 경로를 정의합니다. (예: /diagnose, /users)
 views/: 사용자에게 보여줄 HTML/EJS 템플릿 파일이 들어갑니다. (서버 사이드 렌더링 시 사용)
 public/: 이미지, CSS, 클라이언트용 자바스크립트 등 정적 파일을 보관합니다.
 models/: MySQL DB와 통신하는 로직을 분리하여 관리합니다. (데이터 구조 정의)

**git pull**을 통해 동일한 폴더 구조와 소스 코드를 가지게 됩니다.
코드를 받은 후, 터미널에서 **npm install**만 입력하면, 
프로젝트 폴더 내의 package.json 파일을 읽어 설치한 것과 똑같은 라이브러리들을 자동으로 설치됩니다.

.env 파일은 각자의 로컬 PC 비밀번호에 맞게 개별적으로 수정해서 사용합니다. 
본인만의 MySQL 비밀번호를 입력해 쓰고, ()님은 본인의 비밀번호를 입력해 쓰는 방식입니다.
